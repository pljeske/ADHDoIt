= ADHDoIt
:toc: left
:toc-title: Contents
:toclevels: 2
:icons: font

ADHD-friendly multi-user todo list application. Features smart views, overdue overflow (past-deadline todos are never lost but don't clutter today's list), date/time reminders via email and Web Push, categories, and flexible sorting.

== Features

=== Core

* *Smart views* — Today, Upcoming, Overdue, Done, and per-Category
* *Overdue section* — overdue items surface in a collapsible section on the Today view, keeping the main list clean without losing anything
* *Priority levels* — None / Low / Medium / High with colour-coded indicators
* *Reminders* — opt-in email and Web Push notifications via a scheduled job queue (River on Postgres — no Redis needed)
* *Categories* — colour-tagged groupings with per-category views

=== ADHD UX

* *Quick Capture bar* — inline input at the top of every list; press Enter to add instantly without opening a form
* *Reschedule quick presets* — Tomorrow / Weekend / Next week pills above the date picker; reschedules the deadline without hiding the todo
* *Overdue badge* — live count in the sidebar so nothing slips out of view
* *Today progress counter* — "X tasks today" that updates to "X / Y done" as you work
* *Completion micro-celebration* — brief ring + checkmark animation when a task is marked done

== Architecture

[cols="1,2"]
|===
|Layer |Technology

|Backend
|Go 1.26 — stateless REST API (chi router, pgx/v5, River job queue)

|Frontend
|React 18 + Vite + TypeScript (TanStack Router, TanStack Query, Zustand, shadcn/ui + Tailwind CSS v4)

|Database
|PostgreSQL 16

|Reverse Proxy
|Caddy 2 (automatic TLS in production, SPA fallback, API proxy)

|Job Queue
|River (PostgreSQL-backed, no Redis required)
|===

== Getting Started

=== Prerequisites

* Docker and Docker Compose (or Podman Compose)
* Go 1.23+ (for local backend development)
* Node.js 22+ (for local frontend development)

=== Quick Start with Docker

[source,bash]
----
# 1. Copy and edit the environment file
cp .env.example .env

# 2. Set required secrets in .env:
#   DB_PASSWORD, JWT_SECRET (min 32 chars)
#   SMTP_* and VAPID_* are optional — see below

# 3. Start everything
docker compose up
----

The app will be available at http://localhost:8080.

[NOTE]
====
The `DOMAIN` variable defaults to `localhost`. Caddy runs over plain HTTP on port 8080 in the default configuration.
Remove the `http://` prefix from `frontend/Caddyfile` and expose ports 80/443 to enable automatic Let's Encrypt TLS.
====

=== Local Development (without Docker)

==== Start the database

[source,bash]
----
docker compose up db
----

==== Backend

[source,bash]
----
cd backend
cp ../.env.example .env   # edit DATABASE_URL to point to localhost:5432
go run ./cmd/server
----

The API listens on http://localhost:8080.

==== Frontend

[source,bash]
----
cd frontend
npm install
npm run dev         # dev server at http://localhost:5173
npm run build       # type-check + production build (faster than docker build for TS errors)
----

====
Generate VAPID keys once and keep them in your `.env`. Rotating keys invalidates all existing push subscriptions.
====

== API Reference

Base path: `/api/v1`. All authenticated endpoints require `Authorization: Bearer <access_token>`.

=== Auth

[cols="2,3,3"]
|===
|Method + Path |Request body |Response

|`POST /auth/register`
|`{ email, name, password, timezone }`
|`201` — `{ user, access_token, refresh_token }`

|`POST /auth/login`
|`{ email, password }`
|`200` — `{ user, access_token, refresh_token }`

|`POST /auth/refresh`
|`{ refresh_token }`
|`200` — `{ access_token, refresh_token }`

|`DELETE /auth/logout`
|`{ refresh_token }` _(authenticated)_
|`204`
|===

Refresh tokens are rotated on every `/auth/refresh` call.

=== Todos

[cols="2,3"]
|===
|Method + Path |Notes

|`GET /todos`
|Query params: `view=today\|upcoming\|overdue\|done\|category`, `category_id=<uuid>`

|`POST /todos`
|Fields: `title` (required), `description`, `category_id`, `deadline` (YYYY-MM-DD), `priority` (0–3), `reminder_at` (RFC 3339)

|`GET /todos/:id`
|

|`PATCH /todos/:id`
|Partial update of any create field; changing `reminder_at` cancels and re-schedules the reminder job

|`DELETE /todos/:id`
|Hard delete — `204`

|`POST /todos/:id/snooze`
|Body: `{ snooze_until: "YYYY-MM-DD" }` — reschedules `deadline` to the given date; `status` stays `active`

|`POST /todos/:id/done`
|Sets `status=done`, records `done_at`

|`POST /todos/:id/reopen`
|Clears `done_at`, sets `status=active`
|===

[NOTE]
====
`status` is not directly settable via `PATCH /todos/:id`. Use the dedicated action endpoints (`/snooze`, `/done`, `/reopen`) to maintain invariants.
====

=== Categories

[cols="2,3"]
|===
|Method + Path |Notes

|`GET /categories`
|

|`POST /categories`
|`{ name, color }` — color is a hex string, e.g. `#6366f1`

|`PATCH /categories/:id`
|`{ name, color }`

|`DELETE /categories/:id`
|Todos in this category get `category_id = NULL` — `204`
|===

=== Push Subscriptions

[cols="2,3"]
|===
|Method + Path |Notes

|`POST /push/subscribe`
|`{ endpoint, keys: { p256dh, auth } }` — upserts on endpoint

|`DELETE /push/subscribe`
|`{ endpoint }`
|===

== Key Design Decisions

Stateless backend:: No session storage. JWTs for access tokens, rotating opaque refresh tokens stored as SHA-256 hashes in Postgres.

Overdue is derived, never stored:: `deadline < today AND status = active`. The Today view fetches both today's and overdue todos; overdue items appear in a collapsible section.

`status` via action endpoints only:: `PATCH /todos/:id` cannot set `status` directly. Use `/snooze`, `/done`, `/reopen` to maintain invariants. `/snooze` reschedules the deadline without touching `status` — the todo stays active and reappears in Today or Upcoming on the new date.

Deadlines are dates, not timestamps:: Avoids timezone confusion. "Today" is resolved server-side using the user's stored `timezone` field.

River on Postgres:: Job queue shares the application database. No Redis or separate queue infrastructure needed.

Hard deletes only:: The `done` status acts as the archive. No soft-delete complexity.

Default reminder at creation + 1 h:: Both the form sheet and Quick Capture automatically set `reminder_at` to one hour from now. Users can clear or adjust before saving.

== Production Deployment

[source,bash]
----
DOMAIN=todo.example.com docker compose up -d
----

[IMPORTANT]
====
Before deploying publicly:

. Remove the `http://` prefix from `frontend/Caddyfile` (it disables auto-HTTPS in the default local config).
. Expose ports 80, 443, and 443/UDP (HTTP/3) instead of 8080.
. Set `ENVIRONMENT=production` so the Go server uses JSON structured logging and does not load a local `.env` file.

Caddy will automatically obtain and renew a Let's Encrypt certificate once the domain resolves to your server.
====
